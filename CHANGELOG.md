# Changelog

## 0.3.0 - 2026-07-18

- Makes the public README a concise installation-first entry: one product
  proposition, a Mermaid evidence-before-action flow, an explicit
  repository-local walkthrough, one inspectable Evidence v2 result, and
  repository mapping before optional Graph launch.
- Adds repository-local installation, update, verification, and uninstall for
  Atlas Core with optional Atlas Graph.
- Preserves admitted repository source evidence beside richer consumer
  catalogs, so Evidence v2 retains inspectable source paths and line locators.
- Documents the complete source-admission, digest, indexing, scoring,
  query-time verification, evidence-state, and repository-mapping machinery
  with repository-neutral examples.
- Organizes the supporting documentation by question: product walkthrough,
  evidence construction, agent behavior, instance lifecycle, system
  boundaries, operation, and provenance.
- Adds `atlas map`, which turns bounded, digest-bound repository evidence
  into a validated responsibility-level system model instead of presenting a
  directory or symbol inventory as architecture.
- Adds source-cited semantic rooms, relationships, flows, uncertainty records,
  stale-model quarantine, and source-inventory fallback for arbitrary
  repositories.
- Adds repository-aware Graph titles, capability-based source tabs, room and
  overview navigation, movable navigation nodes, and source-neutral layout and
  routing for mapped repository projections.
- Adds Evidence v2, explicit Generation Provider contracts, repository-local
  reconciliation, deterministic public installation wrappers, and full
  uninstall receipts.
- Applies Apache-2.0 to Atlas and records the complete OFL-1.1 Roboto and Roboto
  Flex notices and font digests.
- Replaces the exported internal build recipe with a public provenance ledger
  containing only component versions, source revisions, review state,
  compatibility records, and payload integrity digests.
- Removes consumer-specific repository names, topology examples, paths, and
  browser-audit expectations from the public documentation and Graph payload;
  the publication audit rejects those concrete setup markers.
