# Changelog

## 0.7.0 - 2026-07-18

- Renders mapped repository responsibilities, relationships, flows, and
  repository-overview return nodes from source-neutral Atlas projections.
- Shows only provider source categories that contain evidence and uses the
  active repository name in the browser title.
- Makes room-entry and repository-overview nodes movable while keeping
  double-click navigation distinct from drag interaction in Safari and
  Chromium.
- Adds container-local directed-flow gradients, deterministic crossing-aware
  ordering, dominant-hub branch fields, vertical routing air, and bounded
  return lanes for dense arbitrary-repository rooms.
- Preserves the Temporal River layout as a documented compatibility fallback
  while using explicit semantic placement for current Architecture
  projections.
- Adds precomputed view-model support, bounded visibility routing, incremental
  rendering, and cached provider projections so large rooms load and interact
  without repeated full routing work.
- Fixes browser viewport overflow, node-label vertical alignment, source-tab
  availability, room collapse, graph-status theming, and repository navigation
  corner cases.
- Locks the standalone visual identity to the paper-lime theme and removes
  runtime theme rotation and technical route-cost coloring.
- Makes repository-setup containers and browser audits source-neutral: no
  consumer repository names, room inventories, or topology-specific label
  assertions remain in the distributable Graph payload.
