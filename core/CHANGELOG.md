# Atlas Core Changelog

## 0.6.0 - 2026-07-18

- Adds `atlas map` with bounded, source-diverse orientation, full admitted-
  inventory freshness, and validated source-cited repository system models.
- Preserves Core-derived repository source evidence beside richer consumer
  catalogs, keeping Evidence v2 paths, digests, excerpts, and line locators
  without fabricating source authority for navigation summaries.
- Adds a repository-neutral, implementation-exact explanation of how admitted
  files become digest-bearing source evidence, local hybrid rankings, verified
  Evidence v2 states, and the separate cited system model used by Graph.
- Consolidates the exported Core documentation so explanatory guides defer to
  one evidence-and-mapping authority while packet, instance, generation,
  architecture, lifecycle, and sync contracts retain their exact scopes.
- Represents responsibilities, relationships, directed flows, and explicit
  unknowns instead of promoting folders, files, classes, or functions into
  architecture rooms.
- Preserves divergent valid model judgments, identifies one current model by
  digest, quarantines stale models, and returns safely to source-inventory
  projection when admitted repository content changes.
- Separates repository authority, semantic-room entry, and repository-overview
  return navigation for arbitrary repositories.
- Adds capability-based Graph source availability, repository-owned browser
  identity, deterministic source fallback projections, and inspectable
  visible/hidden projection counts.
- Adds previewable repository-local uninstall, lock-recorded instruction and
  ignore-file receipts, explicit sync-history handling, recovery from a damaged
  installed command, and exact fresh-repository restoration.
- Adds repeatable code-extension admission and removes abandoned install staging
  after retry or failed revision/lock validation.
- Requires an explicit hosted Generation Provider model through
  `ATLAS_OPENAI_MODEL` or `OPENAI_MODEL`.
- Keeps the export build recipe source-only and emits a sanitized public
  component, compatibility, review-state, and payload-integrity ledger.
- Expresses the frozen Evidence v2 compatibility record through its Atlas Core
  implementation boundary without repository-specific ref names; Evidence v2
  packet semantics and its specification digest are unchanged.
- Replaces consumer-specific examples and fallback language in exported Core
  documentation with repository-neutral installation, update, taxonomy, and
  architecture contracts, while preserving frozen wire identifiers.

## 0.5.1

## 0.5.0

- Separates Repository Provider retrieval, Generation Provider artifact
  writing, and Current Agent interactive execution.

  generation paths with source-digest validation and environment-only secrets.

## 0.4.0

- Adds the deterministic allowlisted public exporter, exact-tree audit,
  generated installation wrapper, and code-documentation coverage contract.

## 0.3.0

- Freezes Evidence v2, adds the minimal repository-root agent invocation, and
  publishes the plain-English Atlas walkthrough.

## 0.2.0

- Adds repository-local installation, movable installed commands, independent
  per-repository instances, explicit multi-machine sync, and reconciliation.

## 0.1.0

- Establishes the local repository provider, disposable SQLite retrieval index,
  cited evidence, and provider-projection boundary.
